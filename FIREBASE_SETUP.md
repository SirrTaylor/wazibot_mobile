# Firebase Setup (Required for Push Notifications)

Push notifications are **optional** — the app runs without Firebase.
To enable them, follow these steps:

## Step 1 — Create a Firebase project

1. Go to https://console.firebase.google.com
2. Click **Add project** → name it `WaziBot Mobile`
3. Disable Google Analytics (optional)
4. Click **Create project**

## Step 2 — Add Android app

1. In Firebase Console → **Project settings** → **Add app** → Android
2. Android package name: `com.wazibot.mobile` (match your `applicationId` in `android/app/build.gradle`)
3. Download `google-services.json`
4. Place it at: `android/app/google-services.json`

## Step 3 — Add iOS app

1. In Firebase Console → **Project settings** → **Add app** → iOS
2. iOS bundle ID: `com.wazibot.mobile` (match your Xcode bundle ID)
3. Download `GoogleService-Info.plist`
4. Place it at: `ios/Runner/GoogleService-Info.plist`

## Step 4 — Enable FCM

1. In Firebase Console → **Cloud Messaging** → already enabled by default

## Step 5 — Uncomment Firebase init in main.dart

```dart
// In lib/main.dart, uncomment:
import 'package:firebase_core/firebase_core.dart';
import 'firebase_options.dart'; // generated by FlutterFire CLI

await Firebase.initializeApp(
  options: DefaultFirebaseOptions.currentPlatform,
);
```

## Step 6 — Generate firebase_options.dart (FlutterFire CLI)

```bash
dart pub global activate flutterfire_cli
flutterfire configure
```

This generates `lib/firebase_options.dart` automatically.

## Step 7 — Register FCM token with WaziBot backend

After login, call:
```dart
final token = await NotificationService.getToken();
if (token != null) {
  await api.post('/business/fcm-token', data: {'token': token});
}
```

Add this endpoint to the WaziBot backend to store and use the token
for sending push notifications when orders/customers arrive.

## Without Firebase

The app works fully without Firebase — just without push notifications.
Users will still see in-app data via pull-to-refresh and manual checks.
